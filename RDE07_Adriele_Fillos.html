Branches

São ramificações de um projeto de software ou documento e tem como objetivo o desenvolvimento paralelo de:

  • novas funcionalidades;
  • correção de problemas;
  • alteração de código;

Quando temos um projeto e esse projeto vai ser colocado em um repositório , podemos colocar ele na:

  • Branch Master ou Main - braço principal de um projeto em um repositório.
  • Branch Secundária - braço secundário de um projeto em um repositório.

AS BRANCH SECUNDÁRIAS NÃO FAZEM ALTERAÇÕES NA BRANCH MASTER.
Exemplo: Se na branch-1 for feita alguma alteração e em seguida dado um commit , a branch-master não vai sofrer nenhuma alteração , ou seja , diversas equipes podem trabalhar paralelamente na branch-master.

  • A branch secundária e a main parecem duas pastas diferentes com cópias do mesmo arquivo.
  • Se ocorrerem alterações no banco de dados , no desingn , ou no conteúdo , quando concluídas as modificações , é possível fazer um merge.
  • Merge: É a mesclagem das Branchs Secundárias com a Branch Master .

PASSO A PASSO PARA CRIAR UM NOVO REPOSITÓRIO LOCAL NO APLICATIVO GITHUB:
Clica em: File - New repository - escolhe um nome Name e preenche a lacuna - coloca uma descrição em Description - marca o quadrinho com a opção Initialize this repository with a README - na parte escrita License procura e marca a opção MIT License - por fim clica em Create repository.

  • Para que o repositório vá para o site do "GitHub.com" , cilca Ctrl+p ou em Publish repository , desmarca a opção Keep this code private para o repositório se tornar público e clica em Publish repository novamente.
  • Entra no site do "github.com" e atualiza a página.
  • Volta no aplicativo "GitHub" e clica em Open Visual Studio Code , para abrir o repositório no app VisualStudio.
  • Após abrir o VisualStudio , cria um arquivo HTML , desenvolve o código e salva as alterações.
  • Após salvar , entra no app GitHubDesktop , vê se o código se atualizou , escreve um nome para a alteração realizada e clica em Commit to main para o app guardar a modificação do código.
  • Clica em Push origin para mandar as alterações ao site "github.com".

PASSO A PASSO PARA CRIAR A BRANCH SECUNDÁRIA SEM AFETAR A MAIN:

  • Clica em main - New branch - coloca um nome Name para o branch - clica em Create branch - por fim clica em Publish branch.
  • Após isso clica em Open Visual Studio Code , faz a inserção de conteúdo no código e salva tudo.
  • Após salvar , entra no app GitHubDesktop , vê se o código se atualizou , escreve um nome para a alteração realizada e clica em Commit to (nome criado para o branch secundário) para o app guardar a modificação do código.
  • Clica em Push origin para fazer a sincronização com o repositório remoto.

PASSO A PASSO PARA FAZER O MERGE:

  • Clica no branch main.
  • Depois clica em Branch e logo após em Merge into current branch... , para fazer a junção com o braço corrente.
  • Verifica se aparece "Merge into main" , clica na opção branch-secundária e clica em Merge branch-secundaria into main.
  • Clica em Push origin.

Hospedagem

A etapa de hospedagem de um site é importante porque consolida e finaliza o trabalho de desenvolvimento ,ou seja, qualquer usuário que tenha acesso à internet pode acessar o site.
Para hospedar ,em geral, deve-se pagar o serviço de hospedagem ,porém o github.com oferece um recurso interessante e gratuito que é o GitHub Pages.

  • O GitHub Pages oferece hospedagem gratuita de sites. Basta criar uma conta e um repositório público.
  • A URL será "https://nomeusuario.github.io/repositorio"
